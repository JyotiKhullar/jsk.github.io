Restructuring a Product User Guide for Better Navigation

Context: Information architecture improvement for a charting product user guide.

Problem:

  • The original structure grouped content by individual chart components, which fragmented the learning flow.
  • Related concepts such as chart types, elements, and getting started topics were spread across separate sections.
  • The hierarchy was inconsistent and difficult to scale as more chart types and feature areas were added.
  • Users had to jump between multiple top-level sections to understand one product area fully.

What I changed:

  • Shifted the structure from component-based grouping to a unified product guide model.
  • Grouped topics by user need and documentation intent, such as overview, setup, data binding, chart types, and interaction.
  • Created clearer hierarchy under chart categories to support both discoverability and growth.
  • Brought related content together so users could move through the guide more naturally.

Why the new structure works better:

  • It reduces fragmentation by bringing related content into one unified guide.
  • It supports a more natural user journey: understand the product, get started, explore chart types, then go deeper.
  • It makes the guide easier to expand as new chart types or capabilities are introduced.
  • It improves findability for both users and AI-based retrieval systems by making topic boundaries and relationships clearer.
